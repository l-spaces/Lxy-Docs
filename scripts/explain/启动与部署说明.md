# 启动与部署说明

## 1. 文档主题概述

本文说明当前仓库里几个可执行入口、推荐启动顺序，以及本地和独立服务的部署边界。

## 2. 对应源码位置

- 主应用入口：`/lxy-admin/src/main/java/cn/com/lxy/StartApplication.java`
- WAR 部署入口：`/lxy-admin/src/main/java/cn/com/lxy/WebServletInitializer.java`
- 监控中心入口：`/lxy-extend/lxy-monitor-admin/src/main/java/cn/com/lxy/monitor/admin/MonitorAdminApplication.java`
- SnailJob Server 入口：`/lxy-extend/lxy-snailjob-server/src/main/java/cn/com/lxy/snailjob/SnailJobServerApplication.java`
- 主应用环境配置：`/lxy-admin/src/main/resources/application.yml`
- 本地环境配置：`/lxy-admin/src/main/resources/application-dev.yml`
- 生产环境配置：`/lxy-admin/src/main/resources/application-prod.yml`

## 3. 核心内容梳理

### 3.1 主应用启动

`StartApplication` 是当前仓库里最主要的启动入口。它使用 `SpringApplication` 启动，并设置了 `BufferingApplicationStartup(2048)`。

### 3.2 部署方式

`WebServletInitializer` 说明主应用可以被外部 Servlet 容器承载，不只限于直接执行 `main` 方法。

### 3.3 独立服务

- `lxy-monitor-admin` 是独立的 Spring Boot Admin Server。
- `lxy-snailjob-server` 是独立的 SnailJob Server。

## 4. 关键实现说明

- `lxy-admin` 在 `application-dev.yml` 和 `application-prod.yml` 中都配置了 `spring.boot.admin.client` 与 `snail-job`，它们默认都是 `false`，需要按环境显式开启。
- `lxy-monitor-admin` 自身默认开启 `spring.boot.admin.client.enabled=true`，说明它既是监控中心也是被监控客户端。
- `lxy-snailjob-server` 通过自身的 Spring Boot 配置独立运行，和主应用不是同一个进程。

## 5. 使用方式或示例

本地推荐顺序：

1. 启动 MySQL、Redis 等基础设施
2. 启动 `lxy-monitor-admin` 或 `lxy-snailjob-server`，如果要联调对应能力
3. 启动 `lxy-admin`

常用命令：

```bash
mvn -pl lxy-admin -am spring-boot:run -Pdev -DskipTests=false
mvn -pl lxy-extend/lxy-monitor-admin -am spring-boot:run -Pdev -DskipTests=false
mvn -pl lxy-extend/lxy-snailjob-server -am spring-boot:run -Pdev -DskipTests=false
```

## 6. 注意事项

- `application-dev.yml` 和 `application-prod.yml` 中的数据库、Redis、监控账号、任务服务地址都带有示例值，部署前必须改成真实环境参数。
- `lxy-admin` 默认不强制开启监控客户端和 SnailJob 客户端，避免开发环境误连外部服务。
