# 项目整体架构说明

## 1. 文档主题概述

`Lxy-Spring` 是一个基于 `Java 17`、`Spring Boot 3.5.x`、`Maven` 的后端多模块项目。

仓库的职责分层比较清晰：

- `lxy-admin` 负责主应用装配、启动入口、认证入口和运行期监听。
- `lxy-modules` 负责业务能力实现，例如系统管理、工作流、代码生成、任务示例和演示模块。
- `lxy-common` 负责通用基础设施，例如 Web、Redis、鉴权、MyBatis、租户、日志、加密、文档等。
- `lxy-extend` 负责可独立部署的扩展服务，例如 Spring Boot Admin Server 和 SnailJob Server。

## 2. 对应源码位置

- 根聚合与版本管理：`/pom.xml`
- 主应用入口：`/lxy-admin/src/main/java/cn/com/lxy/StartApplication.java`
- 主应用装配：`/lxy-admin/pom.xml`
- 公共能力聚合：`/lxy-common/pom.xml`
- 业务模块聚合：`/lxy-modules/pom.xml`
- 扩展服务聚合：`/lxy-extend/pom.xml`

## 3. 核心内容梳理

### 3.1 技术栈

从根 `pom.xml` 可以确认的核心技术栈包括：

- Spring Boot 3.5.12
- MyBatis / MyBatis-Plus
- dynamic-datasource
- Sa-Token + JWT
- Redisson / Lock4j
- springdoc-openapi + therapi javadoc
- Warm-Flow
- SnailJob
- JustAuth
- sms4j
- AWS S3 SDK
- Velocity
- Anyline

### 3.2 模块边界

- `lxy-admin` 只做启动、聚合和入口，不承载共享业务能力。
- `lxy-common` 提供可复用技术能力，避免把通用逻辑写进 `lxy-admin`。
- `lxy-modules` 承载业务实现和示例模块。
- `lxy-extend` 放独立进程，例如监控中心和任务服务。

### 3.3 运行关系

典型运行关系可以概括为：

`lxy-admin` -> `lxy-common-*` -> `lxy-modules/*`

其中几个关键链路是：

- 登录、租户、权限、在线用户：`admin` + `common-satoken` + `common-redis` + `common-tenant`
- 数据访问：`common-mybatis` + `common-tenant`
- 工作流：`workflow` + `warm-flow`
- 任务：`job` + `common-job`
- 监控：`monitor-admin` 作为独立 Server，`admin` 作为 Client

## 4. 关键实现说明

- `StartApplication` 使用 `BufferingApplicationStartup(2048)`，说明启动阶段有基础性能观测。
- `lxy-common-bom` 作为依赖管理层，不是运行时功能模块。
- `lxy-common-job`、`lxy-monitor-admin`、`lxy-snailjob-server` 都是可独立部署的进程单元。

## 5. 使用方式或示例

主应用启动：

```bash
mvn -pl lxy-admin -am spring-boot:run -Pdev -DskipTests=false
```

如果只验证聚合是否能编译：

```bash
mvn -pl lxy-admin -am test -Pdev -DskipTests=false
```

## 6. 注意事项

- 根 `pom.xml` 默认跳过测试，验证时必须显式传 `-DskipTests=false`。
- `lxy-admin` 的默认 Maven profile 是 `dev`。
- 项目不是单体大杂烩，不能把共享能力误写进 `lxy-admin`。
