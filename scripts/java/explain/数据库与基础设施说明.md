# 数据库与基础设施说明

## 1. 文档主题概述

本文说明当前仓库提供的 SQL 初始化脚本和本地基础设施编排文件，以及它们对应的模块关系。

## 2. 对应源码位置

- 数据库脚本目录：`/script/sql`
- Docker 编排目录：`/script/docker`
- 系统库脚本：`/script/sql/ry_vue_5.X.sql`
- 工作流脚本：`/script/sql/ry_workflow.sql`
- 任务脚本：`/script/sql/ry_job.sql`
- Docker 主编排：`/script/docker/docker-compose.yml`
- Docker 辅助编排：`/script/docker/database.yml`

## 3. 核心内容梳理

### 3.1 数据库脚本

- `ry_vue_5.X.sql` 对应系统管理相关数据表。
- `ry_workflow.sql` 对应 Warm-Flow 工作流相关表。
- `ry_job.sql` 对应 SnailJob 相关表和初始化数据。

### 3.2 本地基础设施

`script/docker` 下的编排用于本地联调 MySQL、Redis、MinIO、Nginx、监控中心和 SnailJob 等基础设施。

### 3.3 模块依赖

- `lxy-system` 依赖系统库脚本
- `lxy-workflow` 依赖工作流脚本
- `lxy-job` / `lxy-common-job` 依赖任务脚本和 SnailJob 服务

## 4. 关键实现说明

- `lxy-admin` 的开发和生产配置都直接引用了 MySQL、Redis 和监控/任务服务的地址，说明基础设施不是“可选插件”，而是运行前提。
- `lxy-common-job` 通过 SnailJob 客户端接入调度，任务脚本里的表结构和组配置需要与服务端一致。

## 5. 使用方式或示例

常见联调顺序：

1. 导入 `ry_vue_5.X.sql`
2. 需要工作流时再导入 `ry_workflow.sql`
3. 需要任务能力时再导入 `ry_job.sql`
4. 启动 `script/docker` 中对应的基础设施
5. 启动主应用或独立服务

## 6. 注意事项

- `application-dev.yml` 和 `application-prod.yml` 中使用的是示例 IP 和密码，不能直接作为正式环境配置。
- 工作流和任务脚本不是“可有可无的演示数据”，它们会直接影响对应模块是否能启动或正常访问。
